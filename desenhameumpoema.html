“Desenha-me um poema.” allows the creation of visual poetry by multiple people simultaneously.

The main goal of “desenha-me um poema.” is to allow the creation of an everchanging cybertext in a co-operative way.

From a static initial poem, users can change the position and the opacity of each word and punctuation, as well as rotate them. Each element of the original poem, can only be used once in each new visual creation. When there aren’t any more words or punctuations left, the new visual poetry moves to the left, creating a new blank canvas where it used to be.

The Interface

With this project we wanted to promote the user’s exploration, so the interface aims to be very unclear about how to use it. There are three handlers that have the same visual look, but each one performs a different action: one controls the size of the word, the other its opacity and the last one rotates it.

It was important to set limits to the changes the user can make to each word and punctuation to assure their readibility. If the user could set the opacity to 0, for example, during the creation of the next visual poetry the users would have a hard time finding it, and the never ending creation of visual poetry through this platform could be compromised.

Create Together

The main goal of the project was to allow the creation of visual poetry by multiple people at the same time. Therefore it was necessary to deploy it through a node.js server to keep the construction in process updated in real time.

What others created

Since the project aims to promote an experimental experience and not the production of a final artifact, the different visual poetries created aren’t saved, having an ephemeral existence. These examples were collected by visiting the website and printscreening them as a proof of concept.
